# META TAGS

---

```html
<html lang="en" dir="ltr" xmlns:v="urn:schemas-microsoft-com:vml" xmlns:o="urn:schemas-microsoft-com:office:office">
```

---

### `lang="en"`

* **Purpose:** Specifies the primary language of the document content (`en` = English).
* **Importance:**

  * Signals to email clients and processors that content is in English.
  * Screen readers for visually impaired users rely on this for correct pronunciation and linguistic rules—**critical for accessibility**.
  * Guides spell-checkers and dictionary lookups.
  * Assists translation services in identifying the source language.

---

### `dir="ltr"`

* **Purpose:** Specifies text direction for the content.
* **Details:**
  `ltr` = left-to-right (default for English and most Western languages).
  `rtl` (right-to-left) would be used for Arabic, Hebrew, Persian, etc.
* **Importance:**
  Ensures correct text flow, prevents rendering issues in clients that might try to guess direction.

---

### `xmlns:v="urn:schemas-microsoft-com:vml"`

* **Purpose:** Declares an XML namespace for VML (Vector Markup Language).
* **Details:**
  The `v` prefix associates elements/attributes with the namespace URI.
* **Importance:**

  * Used in older HTML docs, especially those generated by Microsoft Office applications (Word, Excel, etc.).
  * Allows embedding VML graphics for Outlook compatibility (important for background images/layouts).

---

### `xmlns:o="urn:schemas-microsoft-com:office:office"`

* **Purpose:** Declares an XML namespace for Microsoft Office-specific elements.
* **Details:**
  The `o` prefix is associated with the Office namespace URI.
* **Importance:**

  * Found in HTML from Microsoft Office apps.
  * Allows embedding Office-specific metadata or formatting.

---

```html
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
```

---

### `http-equiv="Content-Type"`

* **Purpose:** Simulates an HTTP header, tells the browser/email client how to parse and display content.
* **Importance:**
  In emails (no server), this meta tag is read directly to tell the client: “This is HTML, and here’s the character encoding.”

#### `content="text/html; charset=utf-8"`

* **Details:**

  * `text/html`: Declares the MIME type as HTML.
  * `charset=utf-8`: Specifies UTF-8 encoding.
* **Importance:**

  * **Critical:** Prevents garbled text (mojibake) from incorrect encoding.
  * UTF-8 supports all languages, special characters, symbols, and emojis.
  * Ensures display consistency for all recipients.

---

```html
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
```

---

### `http-equiv="X-UA-Compatible"`

* **Purpose:** Mimics the X-UA-Compatible HTTP header; influences which rendering engine is used in IE/Outlook.
* **Importance:**

  * Relevant for Outlook, which uses the MS Word rendering engine (similar to old IE versions).
  * Tries to make Outlook use the most modern rendering mode it supports.

#### `content="IE=edge"`

* **Purpose:** Instructs to use the highest available rendering mode.
* **Importance:**

  * Avoids falling back to "quirks mode" (old, problematic rendering).
  * Helps maintain layout integrity.

---

```html
<meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=yes" />
```

---

* **Purpose:** Enables responsive layouts, especially on mobile devices.
* **Key Effects:**

  * Tells clients to use the device’s real width.
  * Ensures initial zoom is correct.
  * Allows users to zoom for accessibility.

#### `width=device-width`

* **Sets viewport width to match device’s screen width.**

#### `initial-scale=1`

* **Ensures no zoom-in/out by default; 100% size.**

#### `user-scalable=yes`

* **Allows pinch-to-zoom (improves accessibility and user experience).**

---

```html
<meta name="format-detection" content="telephone=no, date=no, address=no, email=no, url=no" />
```

---

* **Purpose:** Prevents mobile clients (mainly iOS) from auto-linking or styling phone numbers, dates, addresses, emails, URLs.
* **Importance:**
  Prevents unwanted blue links and underlines that can ruin your design.

#### Controls:

* **`telephone=no`**: Stops phone numbers from becoming clickable.
* **`date=no`**: Stops auto-linking of date/time.
* **`address=no`**: Stops addresses linking to maps.
* **`email=no`**: Stops email addresses becoming mailto links.
* **`url=no`**: Stops plain URLs becoming links.

---

```html
<meta name="x-apple-disable-message-reformatting" />
```

---

* **Purpose:** Attempts to stop Apple Mail/iOS Mail from reformatting the message.
* **Importance:**

  * Prevents unwanted font/size/layout changes.
  * Maintains your intended design, especially on Apple devices.
  * **Non-standard, proprietary.** May not always work.

---

```html
<meta name="color-scheme" content="light dark" />
```

---

* **Purpose:** Tells supporting clients you handle both light and dark mode in your CSS.
* **Importance:**

  * Prevents email clients from auto-inverting colors.
  * Allows for CSS `@media (prefers-color-scheme: dark)` rules.

#### `content="light dark"`

* Indicates support for both color schemes (order: light first is standard).

---

```html
<meta name="supported-color-schemes" content="light dark" />
```

---

* **Purpose:** Further signals dark mode support, especially for Gmail and Outlook.com.
* **Importance:**

  * Provides best compatibility across clients.
  * Tells clients not to auto-invert; you provide styles.

---

```html
<title>HTML Email Template Project</title>
```

---

* **Purpose:** Sets the document’s title (seen in browser tabs, bookmarks, and sometimes used as preview text in email clients).
* **Importance:**

  * Used by screen readers for accessibility.
  * Sometimes used as fallback preview/summary if preheader text is missing.
  * Used when emails are opened in new tabs or saved as files.

```html
<!--[if mso]>
  <noscript>
    <xml>
      <o:OfficeDocumentSettings>
        <o:PixelsPerInch>96</o:PixelsPerInch>
      </o:OfficeDocumentSettings>
    </xml>
  </noscript>
<![endif]-->
```

---

* **Purpose:** Fixes DPI scaling issues in older Outlooks.
* **Importance:**

  * Ensures images render at intended size regardless of Windows DPI setting.
  * Maintains pixel-perfect layout, avoids blurry or oversized images.

---

## About Outlook Conditional Comments

* Only read by Outlook (MS Word rendering engine) and old IE.
* Lets you add Outlook-specific code without affecting other clients.

## About `<noscript>`, `<xml>`, `<o:OfficeDocumentSettings>`, `<o:PixelsPerInch>`

* `<noscript>`: Used here to “hide” XML for non-Outlook clients.
* `<xml>`, `<o:OfficeDocumentSettings>`: Proprietary Microsoft tags for Office-specific settings.
* `<o:PixelsPerInch>96</o:PixelsPerInch>`: Sets fixed DPI for consistent image sizing.

# CSS RESETS

---

```html
<style type="text/css">
  /* Reset styles go here */
</style>
```

* **Purpose:** Embeds CSS in the `<head>` for compatibility with strict/old email clients.
* **`type="text/css"`** is kept for compatibility, even though HTML5 makes it optional.

---

## TABLE

```css
table {
  border-spacing: 0;
  border-collapse: collapse;
}
```

* **Purpose:** Removes default spacing and “double borders” from tables.

### `border-spacing: 0;`

* Removes space between adjacent table cells (avoids unwanted gaps).

### `border-collapse: collapse;`

* Merges adjacent cell borders into a single border (removes “double borders”).

---

## TD

```css
td {
  padding: 0;
}
```

* **Purpose:** Ensures no default padding in table cells (for precise layouts).

---

## IMG

```css
img {
  border: 0;
}
```

* **Purpose:** Removes unwanted default borders around images (especially linked images).

---

## Gmail Dark Mode Fixes

### `u + .body`

* **Explanation:** Gmail inserts a phantom `<u>` tag before `<body class="body">` in some renderings. This lets you target Gmail-only scenarios.

#### `.gmail-blend-screen`

```css
u + .body .gmail-blend-screen {
  mix-blend-mode: screen;
  background: #000;
}
```

* Used to “un-invert” light-colored images/logos in Gmail dark mode.

#### `.gmail-blend-difference`

```css
u + .body .gmail-blend-difference {
  mix-blend-mode: difference;
  background: #000;
}
```

* Used to “restore” dark-colored images/logos that Gmail might invert.

## ExternalClass

```css
.ExternalClass {
  width: 100%;
}
.ExternalClass,
.ExternalClass p,
.ExternalClass td,
.ExternalClass div,
.ExternalClass span,
.ExternalClass font {
  line-height: 100%;
}
```

* **Purpose:**

  * Ensures full width and normal line height in Outlook.com and some versions of Outlook.
  * Fixes unwanted line spacing and shrunken widths.

---

## Apple Data Detectors

```css
a[x-apple-data-detectors="true"] {
  color: inherit !important;
  text-decoration: none !important;
}
```

* **Purpose:** Removes Apple’s blue/underlined styling for auto-linked data (phone, date, etc.).

# MEDIA QUERIES

---

## `@media` Rule

```css
@media screen and (max-width: 599.98px) { ... }
@media screen and (max-width: 499.98px) { ... }
@media screen and (max-width: 399.98px) { ... }
```

* **Purpose:** Applies styles only below certain breakpoints for responsive design.

### Why `.98px` instead of `.00px`?

* Prevents sub-pixel rounding bugs at breakpoints—guarantees a clean separation.

# DARK MODE

---

## `:root`

```css
:root {
  color-scheme: light dark;
  supported-color-schemes: light dark;
}
```

* **Purpose:** Declares both light and dark mode support at the document root.

---

## `@media (prefers-color-scheme: dark)`

```css
@media (prefers-color-scheme: dark) {
  /* Dark mode styles here */
}
```

* **Purpose:** Applies rules only for users whose systems/clients prefer dark mode.

---

## `[data-ogsc]`

* **Purpose:** Targets Gmail’s web-client-specific attribute for forced dark mode processing.
